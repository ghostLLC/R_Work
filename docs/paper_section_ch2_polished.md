# 第2章 数据背景与预处理

## 2.1 TCGA数据库与BRCA数据集概述

### 2.1.1 TCGA项目简介

癌症基因组图谱（The Cancer Genome Atlas，TCGA）是美国NCI和NHGRI在2006年联合发起的一项大型癌症基因组研究计划。这个项目跑了十多年，覆盖了33种癌症类型、超过20,000例肿瘤样本和配对正常组织，系统性地生成了基因组、转录组、表观基因组和蛋白质组多个层次的分子数据。到今天为止，TCGA仍然是癌症生物信息学领域最常用的公共数据资源之一——可以说，过去十年里相当一部分癌症基因组学的重要发现，都不同程度地受益于这个项目所积累的数据。

本研究聚焦的乳腺浸润性癌（BRCA）是TCGA最早启动的癌种之一，也是样本量最大的实体瘤项目。BRCA队列涵盖了Luminal A、Luminal B、HER2富集型和三阴性四种分子亚型，分期从I到IV全覆盖。从数据挖掘的角度看，这个队列同时兼顾了"样本量充足"和"临床信息丰富"两个关键条件，对于验证和比较各类分析方法的有效性来说是一个比较理想的数据基础。

### 2.1.2 TCGA数据层级与条形码体系

TCGA对数据做了严格的分层管理。从原始测序文件到可以直接拿来分析的高级数据，大致分成四个级别：Level 1是原始的FASTQ/BAM文件（仅限受控访问）；Level 2是初步处理后的中间数据；Level 3经过了标准化和质控，可以直接用于统计建模——本研究所用的表达矩阵和突变注释文件都属于这个级别；Level 4则是跨样本汇总的综合结果。

TCGA的样本条形码是理解其数据组织方式的一把钥匙。以`TCGA-A8-A079-01A-11R-A002-07`为例：项目代码（TCGA）、组织来源位点（A8）、参与者编号（A079）这些字段拼在一起，构成了样本的唯一标识。其中最关键的是第14-15位的"样本类型"字段：01代表原发实体瘤，11代表癌旁正常组织。这个简单的编码规则，是所有下游分析中区分"肿瘤"和"对照"的最基础依据。
