# 第2章 数据背景与预处理

## 2.1 TCGA数据库与BRCA数据集概述

### 2.1.1 TCGA项目简介

癌症基因组图谱（The Cancer Genome Atlas，TCGA）是由美国国家癌症研究所（NCI）与国家人类基因组研究所（NHGRI）于2006年联合启动的标志性大型癌症基因组研究项目。该项目历时十余年，覆盖33种癌症类型，累计纳入超过20,000例原发性肿瘤样本及其配对的正常组织样本，旨在通过大规模多组学数据整合揭示癌症发生的分子机制。TCGA为每一种癌症类型系统性地生成了基因组、转录组、表观基因组及蛋白质组等多层次分子数据，构建了迄今为止最为全面的癌症多组学公共数据资源库。

乳腺浸润性癌（Breast Invasive Carcinoma，BRCA）是TCGA项目中最早启动的癌种之一，也是样本量最大的实体瘤项目之一。BRCA队列涵盖了不同分子亚型（Luminal A、Luminal B、HER2富集型、基底样型/三阴性）、不同临床分期（I–IV期）以及多民族背景的患者，为乳腺癌的分子分型、预后评估和靶向治疗研究提供了丰富的数据基础。

### 2.1.2 TCGA数据层级与条形码体系

TCGA数据的组织采用严格的分层架构。从原始数据到可供下游分析的高级数据，其处理层级划分如下：

- **Level 1**：原始测序数据（FASTQ/BAM文件），仅供受控访问。
- **Level 2**：初步处理后的中间数据（如未归一化的计数矩阵）。
- **Level 3**：经过标准化和质控的处理后数据（如归一化表达值、突变注释文件），可直接用于统计分析。
- **Level 4**：跨样本汇总的综合分析结果。

在本研究中，我们从Genomic Data Commons（GDC）——TCGA的官方数据门户——获取的是Level 3处理级别的数据，确保数据的可重复性和标准化程度。

TCGA样本条形码（Barcode）是其数据管理的核心标识体系，采用标准化的分层命名规则。以典型的TCGA条形码`TCGA-A8-A079-01A-11R-A002-07`为例，各段含义如下：

| 字段位置 | 示例值 | 含义 |
|---------|--------|------|
| 项目代码 | TCGA | 项目名称 |
| 组织来源位点 | A8 | 样本采集机构编号 |
| 参与者编号 | A079 | 唯一患者标识符 |
| 样本类型 | 01 | 01=原发实体瘤，11=癌旁正常组织 |
| 小瓶标识 | A | 样本分装顺序 |
| 分析物类型 | 11 | R=RNA，D=DNA |
| 测序技术平台 | R | 平台代码 |
| 分析批次 | A002 | 实验批次编号 |
| 分析项目 | 07 | 数据加工项目编号 |

其中，前12个字符（如`TCGA-A8-A079`）构成患者级别的唯一标识符，是多组学数据集成的关键锚点。在本研究的跨组学整合过程中，所有组学数据均依据这一患者ID进行对齐，确保每个患者的mRNA表达、miRNA表达和体细胞突变数据能够精准匹配。

### 2.1.3 BRCA多组学数据架构

本研究涉及的BRCA多组学数据涵盖四个核心数据层：

1. **mRNA转录组**（HiSeq RNA-seq平台）：覆盖基因表达全貌，包含蛋白质编码基因和非编码RNA的表达信息，是分析基因表达差异和通路活性的核心数据层。
2. **miRNA转录组**（HiSeq miRNA-seq平台）：聚焦微小RNA的表达谱，为转录后调控研究提供关键依据。
3. **体细胞突变**（全外显子组测序WES，MuTect2变异检测）：揭示肿瘤基因组中的点突变、小片段插入缺失等体细胞变异事件。
4. **临床信息**（TCGA Clinical模块）：涵盖人口统计学特征、病理诊断、TNM分期、分子分型、治疗方案及生存随访等多维度临床变量。

上述四层数据相互补充，为乳腺癌的分子机制挖掘和临床关联分析构建了完整的分析框架。转录组层面揭示表达异常，突变层面追溯基因组改变的根源，miRNA层面补充表观遗传调控信息，而临床数据则为所有分子发现提供表型锚定。

## 2.2 数据获取与整合

### 2.2.1 基于TCGAbiolinks的数据获取流程

TCGAbiolinks是Bioconductor生态系统中专门用于TCGA GDC数据查询、下载和预处理的R/Bioconductor软件包。它封装了GDC的RESTful API，提供从数据检索到导入分析的完整工作流。本研究的所有多组学数据均通过TCGAbiolinks包从GDC API统一获取，版本为GDC Data Release 42.0。

数据获取遵循"查询-下载-导入"的三阶段流水线模式：

**第一阶段：查询（Query）**

构建GDCquery对象，指定项目（project = "TCGA-BRCA"）、数据类别（data.category）、实验策略（experimental.strategy）及样本类型等过滤条件。

- mRNA表达数据：`data.category = "Transcriptome Profiling"`, `experimental.strategy = "RNA-Seq"`, `workflow.type = "STAR - Counts"`
- miRNA表达数据：`data.category = "Transcriptome Profiling"`, `experimental.strategy = "miRNA-Seq"`
- 体细胞突变数据：`data.category = "Simple Nucleotide Variation"`, `data.type = "Masked Somatic Mutation"`, `workflow.type = "MuTect2 Variant Aggregation and Masking"`
- 临床数据：`data.category = "Clinical"`, `data.type = "Clinical Supplement"`

**第二阶段：下载（Download）**

调用GDCdownload函数批量下载查询命中的所有数据文件。为保障网络传输的稳定性，该函数内置断点续传和自动重试机制。所有数据下载至本地缓存目录，便于后续重复使用。

**第三阶段：导入与解析（Prepare）**

通过GDCprepare函数将下载的原始数据文件解析为R中的结构化数据对象。对于表达数据，生成SummarizedExperiment对象，同时包含表达矩阵、基因注释信息和样本元数据；对于突变数据，导入为MAF（Mutation Annotation Format）格式的标准突变注释数据框。

### 2.2.2 多组学数据患者级整合

多组学整合的核心挑战在于不同数据层的样本标识体系存在差异。mRNA数据使用TCGA Aliquot Barcode（完整条形码），miRNA和突变数据也是如此。为实现在患者级别的整合，我们提取每个样本条形码的前12个字符作为患者ID，并依此进行跨组学对齐。

整合策略为取交集（intersection）方式，即仅保留在所有组学数据层中均有对应记录的患者。具体流程如下：

1. 从mRNA表达矩阵中提取样本列名，截取前12字符作为患者ID，去重（同一患者可能对应多个样本小瓶，选取01A原发性肿瘤样本）。
2. 对miRNA表达矩阵执行相同操作。
3. 对体细胞突变数据，提取Tumor_Sample_Barcode列的前12字符。
4. 对临床数据，提取submitter_id列的前12字符（或直接使用patient_id字段）。
5. 计算四个患者集合的交集，确定可纳入整合分析的核心患者群体。

整合结果如下：mRNA表达数据覆盖1,094例肿瘤样本（另含113例癌旁正常组织样本），miRNA表达数据覆盖1,079例患者，两者取交集后获得1,076例共有患者，重叠率达98.4%（即miRNA队列的99.7%和mRNA队列的98.4%均实现匹配），表明数据整合的稳健性良好。进一步纳入突变数据（990例样本，对应983例患者）和临床数据（1,174例患者），最终多组学整合队列规模为1,076例患者，均具备完整的mRNA、miRNA表达谱和可追溯的临床注释。

## 2.3 数据预处理

### 2.3.1 mRNA表达数据的过滤

从GDC获取的原始mRNA表达计数矩阵包含60,660个基因（Ensembl Gene ID），涵盖蛋白质编码基因、长链非编码RNA（lncRNA）、假基因及各类小RNA基因。为降低噪音并聚焦于具有生物学意义的基因，本研究实施低表达过滤策略：

- **过滤标准**：要求基因在至少10%的样本中计数（Raw Counts）不低于10。该阈值综合参考了edgeR和DESeq2等主流差异表达分析工具推荐的独立过滤准则。
- **过滤结果**：60,660个基因中，25,981个基因（42.8%）通过过滤标准并被保留用于后续分析。被过滤的基因主要以低丰度lncRNA和假基因为主，删除这些噪音信号有助于提升统计检验效能（减少多重假设检验校正的负担）。

### 2.3.2 表达数据的归一化：TPM转换

原始Read Counts受基因长度和测序深度双重因素的影响，不适合直接用于跨基因比较和样本间分析。本研究采用TPM（Transcripts Per Million）作为归一化指标。

TPM的计算分为两步：

**第一步：Counts Per Million（CPM）归一化**

对每个样本，将每个基因的Read Counts除以该样本的总Read Counts，再乘以10^6，实现对测序深度的校正：

$$
\text{CPM}_{ij} = \frac{X_{ij}}{\sum_{g} X_{gj}} \times 10^6
$$

其中，$X_{ij}$表示基因$i$在样本$j$中的原始Read Counts。

**第二步：基因长度归一化**

将CPM除以对应基因的Effective Length（有效长度，单位为千碱基，kb），实现对基因长度的校正：

$$
\text{TPM}_{ij} = \frac{\text{CPM}_{ij}}{L_i}
$$

其中，$L_i$为基因$i$的Ensembl基因有效长度。

TPM值具有理想的生物学解释：一个样本中所有基因的TPM值之和相等，且TPM值与样本中该基因的mRNA分子相对丰度成正比，这使得TPM成为跨样本比较的首选归一化方法。

基因长度信息的获取通过Bioconductor包`GenomicFeatures`和Ensembl基因组注释数据库（GRCh38/hg38）实现。利用`biomaRt`包查询每个Ensembl Gene ID对应的外显子总长度作为基因有效长度。对于极少数在Ensembl最新版本（V110）中已退役或更新为Gene Symbol的基因ID，以在该版本中的对应记录进行映射和长度赋值。

### 2.3.3 基因标识符映射：Ensembl ID → Gene Symbol

TCGAbiolinks返回的mRNA表达矩阵默认以Ensembl Gene ID（格式：ENSG00000XXXXXXXXX）作为行标识符。Ensembl ID具有唯一性和版本稳定性优势，但在生物学解读和文献对照方面，Gene Symbol（如BRCA1、TP53）更为直观。本研究通过`org.Hs.eg.db`（人类基因组注释数据库，Ensembl版本对应最新稳定版）建立映射关系。

对于一对多映射（同一Gene Symbol对应多个Ensembl ID），保留在样本间平均表达量最高的Ensembl ID，确保每个Gene Symbol有且仅有一条表达记录。对于无法映射的Ensembl ID（如已退役的基因、某些非编码转录本的独立ID），在保留Ensembl ID作为后备标识的基础上，标注其映射状态。

### 2.3.4 临床数据的清洗与加工

原始的TCGA临床数据包含94个变量，覆盖患者人口统计学、病理诊断、治疗史、随访信息等多个维度。对其进行系统性清洗和精炼：

1. **变量筛选**：依据临床相关性和数据完整度，从94个变量中提取25个核心变量，涵盖：
   - 基本信息：age_at_diagnosis（确诊年龄）、gender、race（种族）
   - 病理特征：ajcc_pathologic_stage（AJCC病理分期）、ajcc_pathologic_t（T分期）、ajcc_pathologic_n（N分期）、ajcc_pathologic_m（M分期）、histological_type（组织学类型）
   - 分子分型：er_status（雌激素受体状态）、pr_status（孕激素受体状态）、her2_status（HER2受体状态）、molecular_subtype（分子亚型）
   - 治疗信息：prior_malignancy（既往恶性肿瘤史）、prior_treatment（既往治疗史）、radiation_therapy（放疗史）
   - 生存信息：vital_status（生存状态）、days_to_last_follow_up（末次随访时间）、days_to_death（死亡时间）

2. **分子亚型分类校正**：在数据清洗过程中，发现原始分子亚型注释存在分类不一致的Bug。根据ER/PR/HER2免疫组化状态和PAM50分类的联合判断进行人工校正，最终确认分子亚型分布为：Luminal A型950例（86.3%）、HER2富集型37例（3.4%）、三阴性（Triple Negative/Basal-like）115例（10.4%）。校正后的亚型分布与大规模流行病学文献的报道比率（Luminal A约70-75%、HER2富集约10-15%、三阴性约10-15%）相比，Luminal A比例略高，提示TCGA BRCA队列在入组时对Luminal A型存在一定的选择偏倚。

3. **临床分期标准化**：AJCC病理分期取Stage I至Stage IV四个类别。将分期字符串中的"A"、"B"、"C"亚分期字母去除并统一为数字分期。最终分期分布为：Stage I期183例（16.6%）、Stage II期651例（58.9%）、Stage III期251例（22.7%）、Stage IV期20例（1.8%）。Stage II期占比最高（近六成），符合乳腺癌的典型分期分布特征。

### 2.3.5 缺失值处理

临床数据中存在不同程度的缺失值，其来源包括部分患者未检测特定指标（如某些中心的HER2 FISH检测缺省）、数据上报不完整等。对缺失值采取分变量类型的填补策略：

- **数值型变量**（如确诊年龄）：采用中位数（median）填补，避免异常值对均值填补的干扰。
- **分类型变量**（如分子亚型、分期信息）：采用众数（mode）填补，以最大类别作为合理推断值。
- **删失数据**（生存信息）：vital_status为"Alive"的患者，其days_to_death为缺失是天然删失，此类缺失不做填补。末次随访时间完整度较高（>95%），缺失的个别记录以中位随访时间填补。

### 2.3.6 miRNA与突变数据的预处理

**miRNA数据预处理**：miRNA表达计数矩阵（1,881个miRNA）采用与mRNA相同的TPM归一化流程。鉴于miRNA的reads通常更为集中（reads主要映射至成熟的miRNA区域），保留了全部1,881个miRNA，未实施额外的低表达过滤，以避免丢失低丰度但具有重要调控功能的miRNA。

**体细胞突变数据预处理**：采用TCGA官方推荐的MuTect2流程生成的Masked Somatic Mutation数据，已通过质量控制并过滤了常见种系变异和测序伪影。分别统计每位患者的突变频率（mutation burden）和突变类型分布，并按基因维度汇总复发突变事件——发现15,413个基因至少在1例患者中发生体细胞突变，前20高频突变基因（包括TP53、PIK3CA、GATA3等）与已知的乳腺癌驱动基因谱高度一致。

## 本章小结

本章系统阐述了BRCA多组学数据挖掘的前置工作，涵盖TCGA数据库的体系架构、基于TCGAbiolinks GDC API的规范化数据获取流程、以及面向下游分析的多层次数据预处理策略。最终构建的多组学整合数据集以1,076例患者为核心，涵盖25,981个mRNA基因、1,881个miRNA和15,413个突变基因，并辅以25个精炼后的核心临床变量。预处理环节严格执行低表达过滤、TPM归一化、基因标识符映射、临床数据清洗和缺失值填补等标准化流程，为后续的差异表达分析、多组学关联挖掘和预后模型构建奠定了高质量的数据基础。
